# Phase 4B 图表方案重构与数据整理

本阶段只用于确定图表方案和整理后续 `ChartBlock` 可读取的数据结构，不修改网页组件、不修改 `story.js`、不实现正式图表。

## 1. 为什么重构图表

原有图表方案偏向占位数字卡，无法支撑《夜晚的另一面》的数据新闻表达。新版图表需要从“单点数字”升级为“城市系统解释”：空间关系、密度、对比、换算、人群网络和城市类型矩阵。

五个图表分别承担不同叙事任务：

- 亮马河：解释一条水岸如何连接消费、公共空间和国际街区。
- 三里屯：解释热闹消费背后的复杂治理现场。
- 剧场 / 演艺：用北京与全国对比说明文化内容供给强度。
- 夜骑 / 回家路：把夜间出行背后的服务人群可视化。
- 八城：把夜经济从北京经验扩展到全国城市类型差异。

## 3. 每个图表的核心问题

## 7. 二次计算口径

| 图表 | 二次计算 | 口径说明 |
| --- | --- | --- |
| 三里屯 | 日均警情年化换算：`80 * 365 = 29200 起/年` | 仅用于理解压力，不作为正式年度统计。 |
| 演艺 | 北京日均演出、观众、票房 | 由北京 2024 年度数据除以 365 得出。 |
| 演艺 | 全国日均演出、观众、票房 | 由全国 2024 年度数据除以 365 得出。 |
| 演艺 | 北京占全国比例 | 北京年度指标除以全国年度指标。 |
| 演艺 | 单场观众与人均票价 | 观众 / 场次、票房 / 观众换算。 |
| 八城 | 城市坐标 | 编辑归纳，用于解释类型，不代表官方评分。 |

## 8. 风险与注释

- 不要把社媒播放量、打卡量当作实际客流。
- 不要把全国或平台级性别数据当作北京夜间专属数据。
- 不要把不同统计口径强行相除，例如“每名民警对应多少游客”。
- 夜间服务人群缺少统一官方口径，不应做总人数外推。
- 八城坐标是编辑归纳，不是官方排名或评分。
- 所有二次计算必须在图表脚注中标注“二次计算”。
- 暂缺数据统一标注 `"待补充"` 或 `null`。
